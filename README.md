# 🖥️ WebDesk - 远程桌面控制

一个轻量级的远程桌面控制解决方案，基于 WebSocket 实现实时屏幕共享和远程控制。

## ✨ 特性

- 🚀 **轻量高效** - 基于差分传输，只发送变化的屏幕区域
- 🌐 **Web 界面** - 无需安装客户端，浏览器即可访问
- 🔒 **密码保护** - 支持密码验证，保护远程访问安全
- ⚡ **低延迟** - 优化的传输协议，局域网延迟低至 1-10ms
- 🎯 **完整控制** - 支持鼠标、键盘、滚轮等全部输入操作
- 📦 **单文件部署** - 可打包为单个可执行文件，开箱即用
- ⚙️ **灵活配置** - 通过配置文件自定义服务器参数

## 📋 系统要求

### 服务端（被控制端）
- Python 3.7+
- Windows / Linux / macOS

### 客户端（控制端）
- 现代浏览器（Chrome、Firefox、Edge、Safari 等）
- 支持 WebSocket 和 Canvas

## 🔧 安装依赖

```bash
pip install websockets mss pillow pynput numpy
```

## 🚀 快速开始

### 方式一：直接运行 Python 脚本

1. **启动服务端**
```bash
python server.py
```

2. **访问 Web 界面**
```
打开浏览器访问：http://localhost:8765
```

3. **开始控制**
- 如果设置了密码，输入密码验证
- 连接成功后即可看到远程桌面画面
- 使用鼠标和键盘进行远程操作

### 方式二：使用打包的可执行文件

1. **打包程序**
```bash
python build_exe.py
```

2. **运行可执行文件**
```bash
# Windows
dist\WebDesk.exe

# Linux/macOS
./dist/WebDesk
```

## ⚙️ 配置说明

在程序同目录创建或编辑 `config.ini` 文件：

```ini
[server]
# 服务器绑定地址 (0.0.0.0 表示监听所有网卡)
host = 0.0.0.0

# 服务器端口
port = 8765

# 访问密码 (留空表示不需要密码)
password = 

[performance]
# 最大帧率
fps = 15

# 瓦片大小 (像素)
tile_size = 64

# JPEG 压缩质量 (1-95)
quality = 75
```

### 配置项说明

| 配置项 | 说明 | 默认值 | 建议值 |
|--------|------|--------|--------|
| `host` | 服务器监听地址 | `0.0.0.0` | 局域网访问用 `0.0.0.0`，本机访问用 `127.0.0.1` |
| `port` | 服务器端口 | `8765` | 1024-65535 之间的任意端口 |
| `password` | 访问密码 | 空（无密码） | 建议设置强密码保护 |
| `fps` | 最大帧率 | `15` | 局域网可设置 20-30，广域网建议 10-15 |
| `tile_size` | 瓦片大小 | `64` | 64-128，越小越精细但开销越大 |
| `quality` | JPEG 质量 | `75` | 50-95，越高画质越好但带宽占用越大 |

## 🎮 使用说明

### Web 界面功能

- **连接/断开** - 建立或断开与服务端的连接
- **刷新** - 强制请求全屏刷新
- **全屏** - 切换浏览器全屏模式
- **记住密码** - 勾选后自动保存密码（仅保存在浏览器本地）

### 快捷键

- `F11` - 切换全屏模式
- `Ctrl+V` - 粘贴文本到远程桌面

### 统计信息

界面右上角显示实时统计：
- **FPS** - 当前帧率
- **Tiles/s** - 每秒传输的瓦片数量
- **BW** - 带宽占用（KB/s）
- **延迟** - 网络往返延迟（ms）

## 🔐 安全建议

1. **设置强密码** - 在 `config.ini` 中配置 `password` 参数
2. **限制访问** - 将 `host` 设置为 `127.0.0.1` 仅允许本机访问
3. **防火墙规则** - 配置防火墙只允许信任的 IP 访问
4. **使用 VPN** - 通过 VPN 建立安全隧道后再使用
5. **定期更新** - 保持依赖库为最新版本

## 📊 性能优化

### 提高流畅度
- 增加 `fps` 值（如 20-30）
- 减小 `tile_size`（如 32-48）
- 提高 `quality`（如 85-95）

### 降低带宽占用
- 减小 `fps` 值（如 10-12）
- 增大 `tile_size`（如 96-128）
- 降低 `quality`（如 60-70）

### 局域网最佳配置
```ini
fps = 25
tile_size = 64
quality = 85
```

### 广域网最佳配置
```ini
fps = 12
tile_size = 96
quality = 65
```

### 核心技术
- **WebSocket** - 实时双向通信
- **差分传输** - 只传输变化的屏幕区域
- **JPEG 压缩** - 减少传输数据量
- **Canvas 渲染** - 高性能画面显示
- **pynput** - 跨平台输入注入
- **mss** - 高性能屏幕截图

## 🐛 故障排除

### 无法连接服务器
- 检查服务端是否正常运行
- 确认防火墙是否允许端口访问
- 验证 IP 地址和端口是否正确

### 画面卡顿
- 降低 `fps` 或 `quality` 参数
- 检查网络带宽是否充足
- 关闭其他占用带宽的程序

### 鼠标/键盘无响应
- 确认服务端已安装 `pynput` 库
- 检查服务端是否有输入权限
- Windows 可能需要以管理员身份运行

### 延迟过高
- 检查网络连接质量
- 尝试使用有线网络代替 WiFi
- 减小 `tile_size` 和 `quality` 参数
